The five paid modules

Each module is a standalone OS. Install only what you need. They compose — RevenueOS reads StrategyOS canvas, WriterOS pulls voice from positioning, EngineerOS builds from requirements modeled in the canvas.

RevenueOS

Pipeline mechanics. Account research, outbound sequences, deal qualification, handoffs, customer success. The full revenue lifecycle from cold prospect to expansion.

EngineerOS

Product specs and code. Module graph builder, type-safe decomposition, test design, deployment, incident response. Engineering agents that understand your full business context.

WriterOS

Awareness, conversion, and retention content from structured specs. ICP-aware drafting, brand voice enforcement, persuasion patterns, batch production with dependency ordering.

StyleOS

Visual foundation. Design tokens, web and infographic systems, accessibility-as-structure, attention engineering, AI-specific UX patterns. Production-grade frontend with rigor.

FoundryOS

The OS that builds OSs. Specifier, composer, tooler — agents that create new agents, skills, hooks, MCP servers, and infrastructure. The meta-layer.

Pricing and availability per module — see each module's listing. New modules ship as they harden.

How it works
# Each OS is a folder of agent and skill files. Drop into Claude Code and run.
$ git clone https://github.com/BellaBe/strategy-os
$ cd strategy-os
$ claude --agent strategist

# Strategist proposes hypotheses. Gap-definer destroys them.
# Narrator derives positioning. Verifier audits. You approve at K-open gates.
# Everything written to filesystem. Nothing hidden in a database.
